# Catalog Policy

The HUB catalog should be public-facing and honest.

## Allowed states

- `production` — publicly usable and release-backed.
- `beta` — usable but still evolving.
- `dev` — source/scaffold present; do not overclaim.
- `prototype` — concept/demo; not production.
- `review` — needs rights, metadata, or quality review before promotion.

## Link policy

Every promoted product should eventually have:

- GitHub repo
- release/latest link if available
- product description
- platform/formats
- status
- related SoundCloud/YouTube/Release Vault route if relevant

No fake downloads. No fake paid unlocks. No fake account claims.
