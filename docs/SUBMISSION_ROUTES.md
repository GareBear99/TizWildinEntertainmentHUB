# Submission Routes for ARC-Neuron / ARC Ecosystem

This document tracks public submission routes for **ARC-Neuron LLMBuilder**, **ARC Language Module**, **ARC-StreamMemory**, **Omnibinary Runtime**, and related ARC-style local AI systems.

The goal is not spam. The goal is targeted, honest placement in lists/directories where ARC actually fits:
local AI, offline LLMs, LLMOps/MLOps, model governance, agent workflows, RAG/visual memory, language tooling, and reproducible AI infrastructure.

## Primary submission rules

1. Submit only where the repo genuinely fits.
2. Prefer PRs to awesome lists when they accept PRs.
3. Use issues first when a list has strict rules, star thresholds, or unclear categories.
4. Keep descriptions short, technical, and verifiable.
5. Do not claim external datasets are already ingested into ARC-Neuron.
6. Do not claim v11.3/wave5 is the incumbent until Gate v2 promotion evidence reproduces.
7. Do not submit the same repo into multiple categories of the same list unless the maintainer asks.
8. Track every submission in a local checklist or GitHub issue.
